290 /****
291 *******************Proper use of IMPORT, LOCAL, and EXPORT ******************
292 
293 The explanation of the LOCAL vs. static is due to the split nature of
294 the construction in C. The static declaration really does two things,
295 first it declares a variable to be static, that is its value is retained
296 from one function call to another, the second effect is that the variable
297 becomes local, that is its access is restricted to the enclosing block.
298 Actually in C all variables declared within a block (brackets) are local,
299 so the second case only applies to external variables in a file. The
300 default is that externals are global (ie known to all files), so that to
301 restrict the scope of an external variable the static is declaration is
302 used. Note that all external variable are by default static and global.
303 The defines EXPORT, IMPORT, and LOCAL were designed to explicitly
304 declare the scope of a given procedure or variable, thus leaving the
305 declaration static to perform its first (primary) function. Thus the
306 declarations should be used as follows
307 
308 EXPORT - any variable or procedure with global scope, this includes
309 all functions to be called outside of their file of definition, as well
310 as any transfile global variables (these are strongly discouraged).
311 
312 IMPORT - any variable or procedure which is defined elsewhere, IMPORT
313 is really just another name for extern and serves exactly the same function.
314 IMPORTED variable are just that, variables or procedures that are imported
315 from somewhere else.
316 
317 LOCAL - any variable or procedure whose scope is to be limited to its
318 file of definition.
319 
320 static - any variable (presumably within a block) whose value is static and
321 will not change between function calls to that block.
322 
323 Even though in C LOCAL and static are the same, we see that the division
324 of this single variable into two types provides a useful documentation
325 function by separating the static declaration from the access restriction.
326 
327 *****/
