88  /* Structure Defining a Regular Rectangular Grid: */
89 /*
90 * Support has been added to support virtual domain buffer zones
91 * as well as possibly nonconstant grid spacings. The storage
92 * for the grid lines and spacings is taken from the single
93 * array glstore. The allocated size of glstore should be
94 *
95 * dim-1
96 * -----
97 * \
98 * \
99 * size_of(glstore) = \ (3*(gmax[i]+ub[i]+lb[i]) + 1)
100 * /
101 * /
102 * /
103 * -----
104 * i = 0
105 *
106 * Which is just the combined length of the edge and dl uni_arrays.
107 * This storage should be mapped out as follows
108 *
109 * edge[0] = glstore + lb[0]
110 * edge[i] = edge[i-1] + gmax[i-1] + ub[i-1] + 1 + lb[i] (i > 0)
111 * dl[0] = edge[dim-1] + gmax[dim-1] + ub[dim-1] + 1 + lb[0]
112 * dl[i] = dl[i-1] + gmax[i-1] + ub[i-1] + lb[i] (i > 0)
113 * center[0] = dl[dim-1] + gmax[dim-1] + ub[dim-1] + lb[0]
114 * center[i] = center[i-1] + gmax[i-1] + ub[i-1] + lb[i] (i > 0)
115 *
116 * This arrangement has the following benefits,
117 *
118 * 1. Storage need only be allocated once for grid line storage
119 * 2. The addresses edge[i] - glstore, center[i] - slstore,
120 * and dl[i] - glstore are absolute numbers independent of processor.
121 * This is exploited in the pointer reconstruction for the topological
122 * grid that is transferred across multiple processors with the
123 * interface.
124 *
125 * Note that in the the case of the topological grid, consistency
126 * requires that glstore be allocated by store().
127 *
128 * All allocation of the edge, center, and dl fields should be done through
129 * provided functions.
130 *
131 * Besides supporting parallel virtual domains, the precomputation
132 * of the grid cell boundaries and centers should provide increased
133 * efficiency in many cases. Also note that the cell centers, edges
134 * and spacings are related by the formulas:
135 *
136 * dl[i][j] = edge[i][j+1] - edge[i][j]
137 * center[i][j] = 0.5 * (edge[i][j+1] + edge[i][j])
138 */
