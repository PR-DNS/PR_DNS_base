313  /*
314  * SURFACE BOUNDARIES. The boundary of a surface consists of
315  * a set of curves gathered into two classes, pos_curves and
316  * neg_curves. A given curve is defined as positive with respect
317  * the surface if its orientation agrees with that of the adjoining
318  * triangles that connect the curve to the surface, negative otherwise.
319  * Triangles are assumed to be oriented in a counter clockwise manner
320  * where observed from the side into which the normal vector points.
321  * Thus if the vertices of a triangle are p[0], p[1], p[2], then the
322  * normal has direction given by the vector cross product
323  * (p[1] - p[0])X(p[2] - p[0]).
324  * This means that a curve is positive with respect to a surface if
325  * each bond on that curve statisfies bond->start = p[i] and
326  * bond->end = p[(i+1)%3] for the triangle connected to that bond.
327  * Similarlly the curve is negative if bond->start = p[i] and
328  * bond->end = p[(i-1)%3]. It is assumed that all triangles on
329  * a surface are oriented consistently.
330  */

889 /*
890 * Node, Curve and Surface Boundary Flags:
891 *
892 * These flags use 6 bits, 2 per coordinate direction.
893 * The boundary flags are stored in bits, the first 2 for the x
894 * coordinate, the next 2 for the y and the third 2 for z. A two bit
895 * flag is thus 0 = INSIDE, 1 = BOTTOM, 2 = TOP, 3 unft_assigned. Thus
896 * the upper x,y,z corner has the binary flag: 101010 = (octal) 022,
897 * and the upper z face has the binary flag: 100000 = (octal) 040.
898 *
899 * In summary, each flag is a binary number of the form:
900 * flag = (zzyyxx), where zz, yy, or xx = 00, 01, 10, or 11 (binary).
901 *
902 * IMPORTANT NOTE: The following macro assumes that the boundary field
903 * is the first element of any of the possible objects to which
904 * hs or hsb can point. The current legal objects are
905 *
906 * SURFACE, CURVE, NODE (2d only), and POINT (1d only)
907 *
908 * The arrangement of data allows the boundary information to
909 * be extracted from these structures without an explicit knowledge
910 * of the spatial dimension. This in turn provides an increase
911 * in run time efficiency.
912 *
913 * Note: In many cases, the specific boundary is not specified. In
914 * fact, as of this writing (9/23/96), only parts of the 3d code use the
915 * mapping described above. The rest of the time, we only care whether
916 * the object is on the boundary or not. A set of macros is provided
917 * below using BDRY_MASK to serve this purpose. They set or check all
918 * six of the lower order bits. This is necessary because the higher
919 * order bits are being used in certain parts of the code. This will
920 * require some care. Some of the bit field declarations may be local
921 * to facilitate development, and due to the fact that they are used
922 * only locally. These fields may need to be preserved globally, however,
923 * which is the reason for the BDRY_MASK macros below. One cannot
924 * simply set theboundary field to YES or NO, as this will not preserve
925 * the higher order bits.
926 */
