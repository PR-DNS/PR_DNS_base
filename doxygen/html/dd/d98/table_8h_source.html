27 /*
28 * Interface Table:
29 *
30 * Records Storage and other information for INTERFACES in use.
31 * Records the information as a linked list of Tables, one for
32 * each Interface.
33 * Storage for the linked list is generated dynamically as
34 * needed. This allows programs to manipulate varying
35 * numbers of interfaces without difficulty.
36 *
37 * Within each Table, a further linked list of
38 * ChunkSize(intfc)-sized blocks of storage is maintained - this is the
39 * actual area where bonds, curves etc are stored for that interface.
40 * Individual Chunks should be large enough so that large parts of an
41 * interface are stored contiguously. The use of Chunks makes it easy
42 * to deal with a dynamically changing interface without incurring major
43 * loss of contiguousness. The lowest level storage allocator
44 * is only called occasionaly - whenever a new Chunk is required.
45 * Thus there is little overhead incurred by the low level storage
46 * scheme. The actual dispensing of pieces of a chunk is done
47 * by the routine store().
48 *
49 * The definition of the Table structure is in file table.h
50 *
51 *
52 * Copyright 1999 by The University at Stony Brook, All rights reserved.
53 *
54 */
